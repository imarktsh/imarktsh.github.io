第一章 Java语言基础

1.Java语言特点
面向对象 跨平台 内置库类 支持web开发 安全性和健壮性 。

2.形参和实参的区别
形参作业范围为方法内部，生命周期与方法调用相同，形参在方法调用时分配内存，调用结束释放
方法调用过程只能将实参赋给形参，因此对形参的改变不会影响实参的值

3.指针是所指内容的地址，引用是某块地址的别称。指针可以执行加减运算，引用不行

4.&和&&的区别
&是按位与操作符，a&b就是把ab转为二进制然后按位与的运算，&符号表示相与1&1=1 1&0=0 0&0=0。a&&b就是a和b同时满足为true，一个不满足则为false。
5.Java语言中给任何对象赋值null都是合法的。

6.Java新特性
jdk1.7增加了switch语句支持String的特性，他的实现原理：通过对case后面的String调用hashCode()方法，得到一个int类型hash值，然后用这个hash值来匹配所有case。
jdk1.7增加了二进制字面量，增加了二进制字面量表示（0B001，ob111），在数字中可以添加分隔符123_456，编译时下划线会被去掉。这样做可以分辨二进制长数字。
jdk1.7增加了fork/join框架来增强对多核处理器并行计算的支持，应用场景为把一个应用分为多个子任务，并且组合多个子任务可以得到最终结果。
jdk1.8增加了lambda表达式的支持允许把一个函数作为一个方法的参数。例：Arrays.sort(people,(People a,People b) -> a.getAge()-b.getAge());
jdk1.8增加了接口的默认方法实现。

为什么要引入默认接口实现呢，是为了实现接口升级。在原有接口设计中，如果接口要升级需要对类进行修改，实现默认方法可以给升级带来便利。
jdk1.8增加方法引用。指可以直接引用Java类或方法。共有下面四种形式：
引用构造方法ClassName::new
引用类静态方法ClassName::methodName
引用特定类的任意对象方法ClassName::methodName
引用某对象的方法instanceName::methodName
此外还有类型推断

7.访问修饰符由大到小：public protect default private

8.面向对象三大特性继承 封装 多态

9.接口和抽象类
接口和抽象类都不能实例化都需要实现。
不同：接口只有定义没有实现，抽象类可以实现
接口关键字为implements 继承抽象类为extends
接口强调功能的实现设计理念的has a关系抽象类强调所属关系 is-a。
接口用于实现常用的功能，便于日后修改删除。抽象类倾向于充当公共类的角色，不适合日后代码的修改。

10.Java提供两种用于多态的机制分别为重载和覆盖。
重载是值一个类里面有多个同名方法，但方法有不同的参数。
覆盖指子类可以覆盖父类的方法。

11.Overload和Override
重载是使用不同参数来区分，覆盖是派生类覆盖基类。
覆盖是子类与父类的关系，而重载是一个类方法之间的关系。
覆盖参数列表相同，重载无要求

11.Java基本类型
float double char byte short int long
原始数据类型的包装类
Float Double Character Byte Short Integer Long Boolean
int和Integer区别
int是基本数据类型默认值0 Integer是Java为int提供的包装类默认值为null。

12.字符串
String是不可变类，StringBuffer是可变类。经常修改的字符串使用StringBuffer。StringBuilder跟StringBuffer类似但不是线程安全的，因此单线程的话StringBuilder快一些。

13.异常处理
一个父类Throwable 两个错误处理类Error和Exception
Error表示程序出现严重错误，不可恢复。
Exception表示可恢复异常，包含两种类型Runtime Exception和Checked Exception 。
Java异常处理使用了多态概念，如果在异常处理 过程中首先捕获了基类，然后在捕获子类那么子类代码块永远不会执行，正确的写法是先捕获子类然后捕获基类
异常组合方式三种
try/catch
try/finally
try/catch/finally
其中try只能有一个finally是可选的最少一个，catch可以有多个，catch可能不会执行finally一定会执行，可以执行清理工作，比如关闭数据库连接。

14.throw和throws
throw用来抛出一个异常
throw（异常对象）;
throw e;
throws是方法抛出异常的声明
throws Exception；
throw出现在函数体throws出现在函数头，他们都是消极的异常处理方式

15.流
流的本质是数据传输，根据数据处理类型的不同可分为字节流和字符流。
字节流以字节8bit为单位，包含两个抽象类InputStream和OutputStream。字节流以字符16bit为单位，包含Reader和Writer。他们最主要的区别是字节流不会用到缓存，而字符流用到了缓存。
BufferReader和BufferWriter是具有缓冲区的Reader/Writer子类，避免了频繁从物理设备读取数据。

16.FileInputStream与FileReader的区别
都是用来读文件的，InputStream用来读取二进制数（字节流），而Reader用来读文本数据即Unicode字符。本质上所有数据都是字节，不同的是字节转化为字符需要指定编码方式，默认采用系统默认编码方式，也可以指定编码方式例如utf-8编码，不指定可能会导致乱码。

17.序列化
序列化是将对象转化为字节序列的过程。序列化需要实现Serializable接口，这个接口里面没有包含任何方法，使用一个输出流例如FileOutStream来构造ObjectOutputStream对象，接着使用对象的writeObject方法就可以将obj对象写出。
序列化的特点：如果一个类能够序列化，那么他的子类也能序列化。
包含static和transient关键字的数据成员都不能被序列化。
子类实现Serializable接口父类没有实现，子类属性可以序列化父类不可以。

1.8网络通信
网络上两个程序通过双向通信连接实现数据的交换，这个双向链路的一段称为Socket套接字。Java中的套接字分为两种面向连接的Socket使用TCP协议和面向无连接的Socket使用UDP协议，任何一个socket通过ip和端口号唯一确定。
